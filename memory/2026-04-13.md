### Localization Issue Identified
Jiyun reported that almost every page shows English instead of Korean even after selecting Korean.

#### Root Cause Found
- Root-level `ko.lproj/Localizable.strings` had only partial translations (~50 keys)
- Resources-level `DailyWrite/Resources/ko.lproj/Localizable.strings` had full translations (120+ keys)
- The Localization.swift extension looks for bundle by language code
- **But both files existed, and Xcode was using the incomplete root-level one**

#### Solution Applied
- Copied complete Korean translations from Resources to root-level `ko.lproj/`
- Updated Localization.swift with improved fallback logic
- Created LocalizationTestView.swift for debugging

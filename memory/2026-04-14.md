## 2026-04-14 23:55 - WriteHabit Font Issue Investigation

### Problem Identified
Korean custom fonts (KoPub Batang/Dotum, Pretendard, Ridi Batang) are not applying in the rich text editor.

### Root Cause Discovery
Font files are **missing from Resources folder** despite being listed in Info.plist:
- KoPubBatang-Regular.otf
- KoPubBatang-Bold.otf  
- KoPubDotum-Regular.otf
- KoPubDotum-Bold.otf
- Pretendard-Regular.otf
- Pretendard-Bold.otf
- RIDIBatang-Regular.otf
- RIDIBatang-Bold.otf

Info.plist correctly references these under `UIAppFonts` key, but the actual .otf files are not present in the DailyWrite/Resources directory.

### Next Steps
User needs to provide or re-add the font files to the Resources folder. The FontManager code is correct and will work once the font files are present.

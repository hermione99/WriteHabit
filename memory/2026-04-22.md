## KeywordBrowseView Friends Query Fix (2026-04-22 21:34-21:35)

### Problem
Friends showing 0 despite having friends in the system.

### Root Cause
- `friendships` collection doesn't exist in Firestore
- Friends are stored as arrays in user documents (`users/{userId}.friends`)
- `acceptFriendRequest()` uses `FieldValue.arrayUnion` to add friend IDs to user documents
- KeywordBrowseView was querying non-existent `friendships` collection

### Solution
- Changed KeywordBrowseView to fetch friends from `users/{userId}` document's `friends` array
- Updated query logic to handle both directions of friendship (bidirectional)
- Removed dependency on `friendships` collection

### Code Changes
```swift
// OLD: Query friendships collection (non-existent)
// NEW: Fetch friends array from user document
let userDoc = try await db.collection("users").document(currentUser.uid).getDocument()
let friendIds = userDoc.data()?["friends"] as? [String] ?? []
```

if essay != nil { return Color(hex: "4A5A30") } // Done

## Keyword System Architecture

### Just-in-Time Generation
- Keywords not pre-created for all dates
- Generated when user opens app on specific date
- Stored in `keywordArchive` collection with date as document ID

### Stable Keyword Algorithm
- `getStableKeyword(for:date)` uses deterministic algorithm
- Same date always produces same keyword
- Allows viewing future/past months without pre-creating all keywords

### Calendar Range
- 2026 March (minimum) → Current month + 1 (maximum)
- Future months accessible but cards locked
